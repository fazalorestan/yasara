# Sprint 0 Execution Plan

## Objective
Move YaSara from prototype to real product repository.

## Scope
1. Freeze product decisions.
2. Establish clean repository layout.
3. Prepare PostgreSQL + SQLite dev compatibility.
4. Create Docker Compose.
5. Create CI/CD skeleton.
6. Define code quality gates.
7. Define Sprint 1 Binance Market Data implementation plan.

## Out of Scope
- No trading execution.
- No UI redesign.
- No AI implementation.
- No signal generation.
- No private exchange keys.

## Acceptance Criteria
- Repository boots locally.
- Backend health endpoint works.
- PostgreSQL connection settings exist.
- Alembic is configured.
- CI pipeline exists.
- Engineering constitution exists.
- Sprint 1 requirements are documented.
